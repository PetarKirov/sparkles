Show changes between commits, commit and working tree, etc.

This command compares different versions of files. The most common uses are:

1. "git diff" shows changes in your working tree that haven't been staged.
2. "git diff --staged" shows changes that are staged in the index.
3. "git diff <commit1> <commit2>" compares two commits.
4. "git diff <branch1> <branch2>" compares two branches.

By default, diff shows the changes in unified diff format, with context lines to help you understand the changes. You can use various options to customize the output format or filter which files are shown.
