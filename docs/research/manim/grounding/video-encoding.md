# Grounding: Video Encoding (frames → file/stream)

> Not published research. Do not link to it from the survey pages.

Status legend: ✓ verified against a local artifact · ≈ paraphrase-verified · 🌐 web-verified (primary source fetched) · ◯ open
Claim types: quote · fact · figure · behavior · exposition

Grounds [`docs/research/manim/video-encoding.md`](../video-encoding.md) — four
integration styles (piped ffmpeg CLI, in-process libav\*, GStreamer, SVT-AV1
direct/IVF). Per-approach license + version + D-binding verdict are `fact` rows.
Internal QA, excluded from the built site (`srcExclude`) and the link checker.

**Hazards recorded by the page (flagged):** MathAnimation's container is
**mislabelled** — IVF/AV1 stream written to a `.mov`-named file, README says
`.mp4` (Q-svtav1-3); its NVENC hardware path is a **0-byte stub** (C-svtav1-2);
`yuv420p` is limited-range + chroma-subsampled, a color-correctness trap
(C-gotcha-1).

## Pixel-format & color gotchas + caching

| #          | Claim (short)                                                                                                                                                                                                                                        | Type       | Source (locator)                                                                                                       | Status |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------- | ---------------------------------------------------------------------------------------------------------------------- | ------ |
| C-gotcha-1 | `yuv420p` (both forks' default) is 4:2:0 chroma-subsampled and, by ffmpeg convention, limited-range (luma 16–235); feeding full-range RGBA without saying so crushes contrast; escape = `libx264rgb`/`rgb32` lossless full-RGB                       | fact       | https://github.com/3b1b/manim/blob/e61ad5c3f9c9ac96cba7a46dffc665c0ec13beea/manimlib/scene/scene_file_writer.py        | 🌐     |
| C-gotcha-2 | Premultiplied vs straight alpha must match the encoder's expectation or edges get dark/bright fringes; WebM/MOV transparency is a different pixfmt not a flag (`yuva420p`+libvpx-vp9, or `qtrle`/`argb`) — cannot get alpha from plain `yuv420p` mp4 | fact       | https://github.com/ManimCommunity/manim/blob/4d25c031ffe71c602e20935afd54a96f33545a6e/manim/scene/scene_file_writer.py | 🌐     |
| C-cache-1  | Content-hash cache keys on render **inputs**, not encoded bytes, so caching is correct even where the encoder is not bit-stable; both forks emit one partial file per `play()` then concat — the physical cache substrate                            | exposition | page synthesis (frame-capture.d probe)                                                                                 | ◯      |
| C-rec-1    | Recommendation: ship the ffmpeg subprocess first (zero ABI), add in-process libav\* via ImportC later, skip GStreamer, skip SVT-AV1/IVF unless AV1-only is required, structure output for caching from day one                                       | exposition | page synthesis                                                                                                         | ◯      |
