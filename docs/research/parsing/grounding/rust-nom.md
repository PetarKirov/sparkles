## Discrepancies

- **#D1 ⚠ (substantive) — `rust-nom.md:332`.** "nom has dedicated bit parsers (`nom::bits`)
  and a `bitvec` integration (added in `7.0`)." The bitvec/`BitSlice` integration was **added
  in 6.0.0** (CHANGELOG.md:420, "## 6.0.0 - 2020-10-31": "bitstream parsing with the `BitSlice`
  type from the bitvec crate") and **removed in 7.0.0** (CHANGELOG.md:185, under 7.0.0: "the
  `BitSlice` input type from bitvec has been moved into the nom-bitvec crate. nom does not depend
  on bitvec now"). So 7.0 is when bitvec was _dropped_, not added.
  **Fix:** "…and (via the `nom-bitvec` crate since `7.0`) a `bitvec` integration" — or "a
  `bitvec` integration (added in `6.0`, split out to the `nom-bitvec` crate in `7.0`)". The
  same parenthetical reappears in Sources/Performance reasoning; correct once.

- **#D2 ⚠ (minor misquote) — `rust-nom.md:152`.** Quoted as verbatim: "(like a number of
  bytes)". Source `src/lib.rs:179-180` reads "(like `take` which uses a number of bytes or
  character as argument)". The condensation sits _inside_ quotation marks.
  **Fix:** restore the exact parenthetical, or move the gloss outside the quotes.

- **#D3 ⚠ (minor mislabel) — `rust-nom.md:215`.** Code-block comment "// nom 8.0: src/traits.rs
  (Parser trait, abridged)". The `Parser` trait is defined in `src/internal.rs:403`, not
  `src/traits.rs` (that file exists but holds other items).
  **Fix:** label `src/internal.rs`.

- **#D4 (cosmetic) — `rust-nom.md:41`.** Quote drops source's inner quotes: page "recognize the
  word HTTP"; source `src/lib.rs:73-74` "recognize the word 'HTTP'". Single-vs-double quotes
  around "take 5 bytes" likewise normalized. Verbatim-fidelity nit; optional.

- **#D5 (minor) — `rust-nom.md:422`** (winnow table "Why" cell). Quoted "`I` need not be
  `Copy`"; source `winnow/src/_topic/nom.rs:144` = "`I` does not need to be `Copy`". The cell
  already uses "…" (condensation), so low severity; tighten if strict verbatim wanted.

- **#D6 (unsourced detail) — `rust-nom.md:55` & `:388`.** "Rust parsers since Suricata 4.0" is
  not in the Chifflier & Couprie paper (no "Suricata 4.0" string); the paper confirms the
  Suricata integration but not the version. General/web fact — fine to keep, but it is not
  backed by the cited local primary.

- **#D7 (attribution = web, not local) — `rust-nom.md:406-408`.** "winnow is Ed Page's fork…
  began life as Page's private `nom8` fork". Local winnow repo names no author (`Cargo.toml`

  (winnow `CHANGELOG.md:1091` links the "`nom8` changelog"), but the "Ed Page"/"private fork"/
