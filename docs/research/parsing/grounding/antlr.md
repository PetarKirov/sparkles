## Discrepancies

- ⚠ **`antlr.md:187`** — lexer quote attributed to _§2.2_. Actual section is **§2.3
  _Lexical analysis with ALL(\*)_** (`P` l.227, `-layout` page-3 header). §2.2 is
  _Left-recursion removal_. **Fix:** change `§2.2` → `§2.3`.

- ⚠ **`antlr.md:219`** — left-recursion quote attributed to _§2.4_. Actual section is
  **§2.2 _Left-recursion removal_** (`P` l.281). **There is no §2.4** in `P` (§2 =
  2.1 Sample grammar, 2.2 Left-recursion removal, 2.3 Lexical analysis). **Fix:**
  change `§2.4` → `§2.2`.

- ⚠ **`antlr.md:9,271,317,329,354,362`** — repeated "ten runtime targets … in
  `runtime/` … all first-party / co-tested." The repo's `runtime/` (pinned
  `7d5770395`) has **8 subdirs**: `Cpp CSharp Dart Go Java JavaScript Python3 Swift`.
  PHP's runtime is a **separate repository** (`doc/releasing-antlr.md:80`: "copy the PHP
  runtime into the ANTLR repository to run the unittests… in the separate repository");
  TypeScript is generated against the **`JavaScript`** runtime
  (`runtime/JavaScript/tsconfig.json`). "ANTLR 4 supports 10 target **languages**"
  (`README.md:24`) is accurate, but the page conflates that with "10 first-party
  runtime subdirectories." **Fix:** line 362 most acute — `[runtime/ — the ten runtime
targets (…PHP, Dart)]` should read "**eight** runtime targets (Java, C#, Python3,
  JavaScript, Go, C++, Swift, Dart)"; elsewhere qualify "10 target languages (8 with
  in-repo runtime dirs; PHP runtime is a separate repo, TypeScript ships with the
  JavaScript runtime)."

- ⚠ **`antlr.md:189`** _(low severity, claim correct)_ — "maximal munch: the rule that
  matches the longest input wins, and ties break by grammar order" is cited to
  `doc/lexer-rules.md`, but that file does **not** state longest-match / tie-by-rule-order
  (it only notes identical-literal ambiguity at :170). The behavior is genuine ANTLR/lex
  semantics but unsupported by the cited locator. **Fix:** drop the precise locator or
  cite the ANTLR book / a doc that actually states it.

- ⚠ **`antlr.md:265,340`** _(minor locator)_ — the O(n⁴) complexity sentence (`P`
  l.93–99) and the testing-burden quote "programmers must cover as many grammar position
  and input sequence combinations as possible" (`P` l.102–103) are attributed to _§1.1_,
  but both sit in **§1** (intro), before the §1.1 _Dynamic grammar analysis_ heading
  (`P` l.120). Quotes themselves are verbatim and Theorem 6.3 is correctly cited. **Fix:**
  `§1.1` → `§1` for these two (the genuine §1.1 quotes at L119–138 are correct).
