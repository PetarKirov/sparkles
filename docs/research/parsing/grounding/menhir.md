## Discrepancies

- **⚠ menhir.md:29** — Intro "verbatim" quote is **misquoted vs the pinned manual**. Page reads
  "…decorated with semantic actions expressed in the OCaml programming language, into parsers,
  again expressed in OCaml. It is based on Knuth's LR(1)…". Pinned `manual.tex:56-58` reads
  "…decorated with ``semantic actions'' (fragments of executable code), into parsers. It is
  based on Knuth's LR(1)…". The "expressed in the OCaml programming language … again expressed
  in OCaml" clause is not in the source (likely an older manual edition or paraphrase).
  **Fix:** replace with the pinned wording, or downgrade from a blockquote to a paraphrase.

- **⚠ menhir.md:142-164** — The standard-library code block is introduced as "**Verbatim** from
  the standard library source" but is **not verbatim**: `front/standard.mly` carries an
  `[@name none]`/`[@name some]`/`[@name nil]`/`[@name cons]`/`[@name one]`/`[@name more]`
  attribute on every branch (added for the `.messages`/`--name` machinery), all omitted on the
  page. **Fix:** either include the `[@name …]` attributes, or change "Verbatim from" to
  "Adapted from" / "Simplified from".

- **⚠ menhir.md:19 / :318** — Mirror link `[mirror]: https://github.com/savonet/mehnir` is
  suspect. "savonet" is the unrelated Liquidsoap organisation and "mehnir" is a misspelling of
  "menhir"; this is almost certainly a wrong/fabricated URL (web-unverifiable from local
  artifacts). **Fix:** drop the mirror claim, or replace with a verifiable mirror (e.g. the
  `LexiFi/menhir` fork already cited on the page, or the official GitLab only).
