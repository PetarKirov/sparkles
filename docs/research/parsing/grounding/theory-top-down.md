## Discrepancies

- ⚠ **`top-down.md:111-112` — GCC/Clang misattributed to the ALL(\*) paper.** Text:
  "(the [ALL(\*) paper][allstar] notes the GCC and Clang C/C++ front-ends and the Java
  compiler `javac` all use hand-written recursive descent)". Full-text grep of
  `allstar` for `gcc`/`clang`/`c++ front`/`production compiler` returns **nothing**; the
  paper references **only** the Java compiler `javac` as a "handbuilt recursive-descent
  … parser" (`allstar` L798, L968). The GCC/Clang fact is true in reality but is **not**
  stated by the cited paper. **Fix:** narrow the attribution to javac, e.g. "(the ALL(\*)
  paper benchmarks against the hand-built recursive-descent parser of the Java compiler
  `javac`; GCC's and Clang's C/C++ front-ends are likewise hand-written RD)" — i.e. move
  GCC/Clang outside the `[allstar]` citation. (The unattributed table row at
  `top-down.md:453` and strengths bullet at `:472` are fine — they make no source claim.)

- ⚠ **`top-down.md:435-436` (and :433) — O(n⁴)-explanation quote location is §1, not §6.**
  The Theorem 6.3 quote ("ALL(\*) parsing of n symbols has O(n⁴) time.") is exact
  (`allstar` L800, §6). But the trailing explanatory quote — "because in the worst-case,
  the parser must make a prediction at each input symbol and each prediction must examine
  the entire remaining input; examining an input symbol can cost O(n²)" — is **verbatim**
  yet sits at the end of **§1** (`allstar` L62–67), where it cites Thm 6.3; it does not
  appear in §6. Page cites it as "(§6)". **Fix:** change the `(§6)` locator on that
  sentence to `(§1)` (or "§1, citing Theorem 6.3"). Low severity — wording and the
  O(n⁴)/O(n²) numbers are correct.

## Unsourced / opinion / secondary-only

- ◯ **Claim 3/5 — Rosenkrantz & Stearns 1970 specifics** (decision procedure, ε-rule
  elimination, deterministic pushdown recognizer, decidable equivalence, strict
  `LL(k)⊊LL(k+1)`). Source is Elsevier-paywalled and unobtainable locally; per
  `_sources.md` these are grounded secondarily (Dragon / `gj`) + bibliographic. The
  strictness result is textbook-standard and corroborated. **Formatting nit, not an
  error:** the page renders the strictness sentence as a block**quote** ("for each value

  verbatim quote, and cannot be verified verbatim against the unobtainable original.
  Consider not styling a paraphrase as a quotation.
