## Discrepancies

⚠ **D1 — Intro quote not in the pinned manual (misquote / stale-edition wording).**
`bison-yacc.md:36` quotes the Bison manual's self-description as:

> "Bison is a general-purpose parser generator that converts a grammar description for an LALR(1) context-free grammar into a C program to parse that grammar. …"
> The pinned `doc/bison.texi:588-590` actually reads:
> "Bison is a general-purpose parser generator that converts an annotated context-free grammar into a deterministic LR or generalized LR (GLR) parser employing LALR(1), IELR(1) or canonical LR(1) parser tables."
> The quoted first sentence is an **older edition's** wording (the page sources it from `[intro-mirror]` = `web.mit.edu/gnu/doc/html`, a stale mirror), not the canonical manual. Only the second sentence ("Bison is upward compatible with Yacc…", `:595-596`) is verbatim in the pinned text.
> **Fix:** replace the first sentence with the pinned wording above (and cite `bison.texi`/the current `gnu.org` manual, not the MIT mirror), or drop the verbatim framing for that sentence.

⚠ **D2 — Dangling-else counterexample block: reduce derivation is not verbatim.**
`bison-yacc.md:151-154` renders the reduce derivation as a single collapsed line:

```
  Reduce derivation
    if_stmt
    ↳ 4: "if" expr "then" stmt "else" stmt
```

The manual's `@danglingElseCex` macro (`doc/bison.texi:8576-8581`) produces a **four-line nested** derivation:

```
  Reduce derivation
    if_stmt
    ↳ 4: "if" expr "then" stmt                                "else" stmt
                           ↳ 2: if_stmt
                                 ↳ 3: "if" expr "then" stmt •
```

The page claims this is "verbatim output" but the reduce derivation drops the nested `↳ 2: if_stmt` / `↳ 3: …•` lines. (Header, shift derivation `:145-150`, and the 2nd `Example:` line all match.)
**Fix:** reproduce the full four-line reduce derivation from `bison.texi:8576-8581`, or soften "verbatim" to "abridged".
