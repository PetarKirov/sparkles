## Discrepancies

- **#D1 ⚠ (substantive) — `haskell-uom-plugin.md:219`.** "reports contradictions (`Lose`,
  e.g. `kg ~ m`) as `TcPluginContradiction` — which is what turns into the user-visible
  type error". At the pin, contradictory **wanteds** are deliberately _not_ reported:
  `Plugin.hs:108` reads `Impossible _eq _ -> return $ PluginAPI.TcPluginOk [] [] -- Don't
report a contradiction, see #22`, so a user's `kg ~ m` mismatch surfaces as GHC's
  ordinary unsolved-equality error (exactly what `ErrorTests.hs` string-matches).
  `TcPluginContradiction` (`reportContradiction`, `Plugin.hs:113-114`) fires only for
  contradictory **givens**.
  **Fix:** step 4 rewritten to distinguish givens (`TcPluginContradiction`) from wanteds
  (left unsolved, GHC reports). Fixed: yes (this pass).

- **#D2 ⚠ (minor misquote) — `haskell-uom-plugin.md:273`.** Quoted as verbatim: defines
  `Convertible u v` as "both units reduce to the same canonical form". Source
  `Convert.hs:145-147`: "Two units are convertible if they are both 'Good' and they have
  the same canonical units (and hence the same dimension)." The paraphrase sat inside
  quotation marks.
  **Fix:** de-quoted the paraphrase and added the verbatim source sentence. Fixed: yes
  (this pass).

- **#D3 ⚠ (minor figure) — `haskell-uom-plugin.md:538`.** The iteration-limit comment
  "appears twice"; it appears **three** times in `Tests.hs` (`:29` under the rejected
  `PartialTypeSignatures` block, `:215` and `:232` for the two GHC-8.0.2-disabled tests).
