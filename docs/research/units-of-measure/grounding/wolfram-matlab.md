## Discrepancies

- **#D1 ⚠ (minor) — `wolfram-matlab.md:150-156` (capture-limitation note).** The note
  claimed "no in-notebook input/output — including any error-message text — exists in
  the local captures". Raw-HTML audit of `wolfram-quantity.html` (and siblings) shows
  example **inputs** _do_ survive as screen-reader-only figcaptions ("Wolfram Language
  code: `Quantity[8, "Meters"]`", plus JSON-LD `SoftwareSourceCode` blocks); it is the
  **outputs** that exist only as image references (`clipboard-output` divs with
  `Files/….png` `data-src`, no text alternative). The page's downstream policy (quote
  no output/error text) was and remains sound. **Fix:** note reworded to distinguish
  inputs (present as captions) from outputs (images only). Fixed: yes (this pass).

- **#D2 ⚠ (minor) — `wolfram-matlab.md:110-115` (usage fence framing).** The fence's
  two usage forms are verbatim, but its parenthetical comments condense the capture's
  usage sentences ("represents a quantity with size magnitude and unit specified by
  unit." / "assumes the magnitude of the specified unit to be 1.") while the lead-in
  labelled everything "verbatim from the reference page". **Fix:** lead-in now says the
  comments condense the capture's usage sentences. Fixed: yes (this pass).

- **#D3 ⚠ (minor) — `wolfram-matlab.md:304-308` (DatedUnit gloss grounding).** The
  gloss "`DatedUnit` (a unit pinned to a date)" was cited to the units-guide capture,
  which only **names** DatedUnit (under "Currency Units") without defining it. The
  capture path for the gloss is the KnownUnitQ capture's Scope examples
  (`DatedUnit["USDollars", 1990]`, `DatedUnit["Euros", Today]`; also UnitDimensions
  P&R "UnitDimensions ignores the date in DatedUnit specifications"). **Fix:** gloss
  reworded to "a unit paired with a date, as in `DatedUnit["USDollars", 1990]`" and
  recited to the KnownUnitQ capture. Fixed: yes (this pass).
