## Discrepancies

- **‚öÝ D-DT1 ‚Äî the event-horizon ASan/TSan recipe is a false green on today's
  dub.** `DFLAGS=‚Ä¶ dub test` (no `-b`) compiles without `-unittest` (magic
  `$DFLAGS` build type), runs zero tests, exits 0 printing dub's success line.
  Anyone following the repo's recorded recipe verbatim today validates nothing.
  The page states the trap in a `> [!WARNING]` and prescribes `-b unittest`; the
  historical-recipe angle is cross-linked to `sparkles-baseline.md`. `[hw]` (L27)
- **‚öÝ D-DT2 ‚Äî `--force` was unnecessary in the event-horizon recipe.** dub's
  build ID hashes dflags, so sanitized builds get their own cache slots. Page
  states `--force` is not required. `[hw]` (L31)
- **‚öÝ D-DT3 ‚Äî no shipped druntime has `SupportSanitizers`, not even the official
  tarball.** Expected for nixpkgs; a surprise for the official release
  (`main.yml` never sets `RT_SUPPORT_SANITIZERS`). Yet the flagship fiber-UAR
  catch does not need it ‚Äî only fiber-switch bookkeeping and fake-stack GC
  scanning do. Page states both halves. `[hw+src]` (L20, L22, L23)
- **‚öÝ D-DT4 ‚Äî dub ‚â• 1.42.0-beta.1 spreads `-fsanitize=` in both directions.**
  The brief's mixed-instrumentation question ("consumer instrumented, prebuilt
  impl not") **cannot arise through dub channels** on a current dub ‚Äî but is the
  default on older dubs, where it is a silent coverage gap (E6), not a link
  error. Page states the version boundary explicitly. `[hw+src]` (L30, L32)
- **‚öÝ D-DT5 ‚Äî the MSan hypothesis is half-right in an unexpected way.** MSan's
  default-path problem is that it never _links_ (gcc fallback), not that it
  crashes; once linked via real compiler-rt it runs and even catches true
  positives in trivial programs. The uninstrumented world manifests as
  split-compilation false positives and GC-memory false negatives. Page's MSan
  section states the three stages. `[hw]` (L47, L48, L49)
- **‚öÝ D-DT6 ‚Äî GDC is not dead for ASan.** Recon's `cannot find -lasan` was a
  packaging gap, not a compiler gap; with `-B`/`-L` and one `--param` it catches
  real bugs. The `asan-globals` ‚Üî `ModuleInfo`-walk conflict is a genuinely new
  D-specific footgun. Conversely, GDC's UBSan is check-empty for D at 11.5. Page
  states both in concern 1. `[hw]` (L38, L39)
- **Note ‚Äî L13 (plain-`ld` path) and L23 (premature-collection hazard) are `‚óØ`.**
  L13 is source-verified but untested (the sparkles configs use `gold`, so it is
  not exercised); L23 is a real source-stated hazard not reproduced in four
