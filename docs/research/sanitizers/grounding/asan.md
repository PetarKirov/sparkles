## Discrepancies

- **‚öÝ D-A1 ‚Äî `detect_leaks=0` + manual `__lsan_do_leak_check()` does NOT compose**
  (register R3). The brief hypothesized this per-test recipe; the manual entry points
  are themselves gated on `detect_leaks` (L33). The page states the working recipe
  (`leak_check_at_exit=0` + recoverable check, L34). `[hw-verified: x86_64-linux]`
- **‚öÝ D-A2 ‚Äî nixpkgs LDC uses GCC's runtime, not a bundled compiler-rt** (register R2).
  The recon hypothesis expected LDC's own `clang_rt`; the reality is the gcc link
  fallback ‚Üí GCC 15.2 libsanitizer (L13/L14). The further recon hypothesis of behavioural
  _drift_ (GCC lacking `__sanitizer_*` entry points) is also refuted ‚Äî the hybrid is a
  strict superset (L15). The page states both. `[hw-verified: x86_64-linux]`
- **‚öÝ D-A3 ‚Äî recovered ASan errors leave exit code 0** (register R25). A continue-mode
  runner cannot read exit codes; it must count reports (L26). Stated in concerns 4 and 6.
  `[hw-verified: x86_64-linux]`
- **‚öÝ D-A4 ‚Äî a green D suite fails at exit under default LSan** (register R24).
  `defaultTraceHandler` leaks turn "278 passed" into exit 1; `detect_leaks=0` or a
  suppression file is mandatory policy (L37). Stated in the LSan section.
  `[hw-verified: x86_64-linux]`
- **‚öÝ D-A5 ‚Äî no `LDC_LeakSanitizer` version identifier.** The brief's "verify exact
  names" turned up a real gap; LSan probes detect instrumentation via `dlsym` (L36).
  Stated in the LSan section. `[source-verified]`
- **Note ‚Äî `Die()` does not flush stdio** (surprise). A report callback's buffered
  `printf` is lost; the handler must write-and-close its own sink (L22). Stated as a
  proven caveat in concern 4. `[hw-verified: x86_64-linux]`

**Net:** 0 substantive discrepancies remaining. Four verbatim primary-source quotes
(ABI stability, the gcc fallback, `log_path` flag text, halt-vs-recover) plus the two LSan
quotes are transcribed from the pinned trees; all five probes reproduced their catches.
The ‚öÝ items are brief-vs-reality corrections (R3, R2, R25, R24) and one real-gap finding
(no LSan version identifier) that the page now states correctly; L38 (root-region
mitigation) is honestly `‚óØ` ‚Äî API-present but not exercised.
