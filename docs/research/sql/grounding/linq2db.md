## Discrepancies

- **#D1 (positioning is grounded, not web).** Every framing slogan the page leads with —
  "fastest LINQ", "one step above micro-ORMs … not with magic strings", "not as heavy as …
  Entity Framework … no change-tracking", "type-safe SQL" — is **local** in `README.md:8-14`,
  not web-attested. The `csproj` (#5) restates it. No web fallback was needed for positioning.

- **#D2 (interpolated-`FromSql` parameterization = inference).** The page states an interpolated
  `FromSql($"… {value}")` lifts each `{value}` into a parameter (#18). `RawSqlString.cs:6-9`
  confirms the overload split between a plain `RawSqlString` and an interpolated
  `FormattableString`, and the parameterize-by-default rule (#14, `Sql/Readme.md:15`) applies;
  the specific "each interpolation hole becomes a bind parameter" behaviour is the documented
  linq2db model but is asserted from the overload design + default-parameterization rule rather
  than a single verbatim sentence in the checkout. Consistent, low-severity inference.

- **#D3 (`InlineParameters` doc is a two-line comment).** The quoted `InlineParameters` text
  (#15) spans `IDataContext.cs:53-54` (the `<summary>` runs onto a second line); the page joins
  the two lines into one sentence. Verbatim modulo the line join.

- **#D4 (no `⚠` factual errors).** No claim contradicts the source. The two "absence"
  findings — no migration runner (#29), no nested transactions/savepoints (#43) — are
  grounded: #43 is a positive throw in code; #29 is a true absence in `Source/LinqToDB/`
  (DDL emit exists, a versioned migration engine does not).

## Unsourced / opinion

- Editorial framings — ".NET cousin of `Quill`'s compile-time quotation", "reifies … **at
  runtime**" (contrast with Quill compile-time; grounded in `ExpressionQuery.GetQuery` running
  per-execution, #9), "one rung above the `Dapper` micro-mapper, below `EF Core`", "persistence
  is a verb, not a flush", "the escape hatch is the opposite of the usual one" (inlining is the
  opt-_out_), and the Strengths / Weaknesses / Key-design-decisions table — are author synthesis
  (◯), each consistent with the verified mechanics above. Not independently citable.
- **"No change tracking / identity map / unit of work"** rests on the explicit README statement
  "There is no change-tracking" (#3) plus an _absence_ argument: no session that snapshots loaded
  POCOs and flushes a diff exists in `Source/LinqToDB/`; every write is a discrete
  `Insert`/`Update`/`Delete`/`Merge` call (#34–#37). True by construction, framed against the
  [concepts ORM-patterns] ladder.
- **"Pooling delegated to ADO.NET"** is an absence argument (no pool implementation in
  `Source/LinqToDB/`; the library opens provider `DbConnection`s and relies on the driver pool),
  consistent with the README's "releasing the database connections back into the pool" (#39).
- First-release date (#50) and the current adoption of the named OSS users (#49) are
  web/point-in-time; the users list itself is in-tree (`README.md:50-52`).
