## Discrepancies

- **#21 (list-flattening).** `PersistenceContext.java` documents the tracked state as an HTML
  `<ul>` (`entities`, `collections`, `snapshots`, `proxies`); the page renders it as an inline
  comma list. Wording verbatim, structure normalized. Non-substantive.
- **#41 (composite example).** The `@Entity class Book { … }` block is a _composed_
  illustration: the bare `@Entity` block is from `Entities.adoc:47-56`, `@Id @GeneratedValue`
  from `Entities.adoc:224`, and `@ManyToOne(fetch=LAZY)` from `Mapping.adoc:314`. Each element
  is grounded, but no single verbatim block in the tree contains all of them together. Flagged
  as illustrative, consistent with the survey's other pages.
- **#50 (typography).** Source list markers use a hyphen ("Proxy generation - runtime …"); the
  page uses an em-dash. Cosmetic.
- **#6 / #63 / #64 (local-mention, web-detail).** The `changelog.md` HEAD (`7.4.0.CR1`),
  "Hibernate Reactive", and "Steve … has led the project" are locally attested, but the
  _"current stable is 7.x"_ characterization, the Reactive **Vert.x/Mutiny** API, the
  relicense **year (2022 / 6.0)**, and **Spring Boot default** status are web-attested and
  marked as such on the page.

## Unsourced / opinion

- Editorial framings — "the archetypal JVM ORM", "the design that every functional data mapper
  in this survey deliberately rejects", "the exact inverse of the functional mappers", "deep
  magic", the Strengths/Weaknesses lists, and the trade-off table — are author synthesis. Each
  is consistent with the verified mechanics above (dirty checking vs explicit changeset, lazy
  proxies vs explicit joins, thrown vs `Result`-valued errors) but is not independently
  citable.
- Peer comparisons to `EF Core` / `SQLAlchemy` / `SeaORM` / `Django ORM` / `doobie` / `Ecto`
  are structural claims about _other_ libraries (grounded in their own pages/ledgers), used
  here only for positioning.
- The "≈ 2001–2002" first-release and the LGPL→Apache relicense are web/point-in-time facts,
  correctly flagged on the page.
