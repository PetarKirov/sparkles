## Discrepancies

- **None material.** All quoted strings match the pinned tree verbatim (Javadoc was reflowed
  from `//!`/`*` line prefixes, `{@link X}`/`<code>` markup was stripped to plain backticks,
  and inner `\"…\"` escapes normalized). Two typographic normalizations flagged inline:
  - **#38** — source `Records with degree &lt;= 22` renders in the page as `degree ≤ 22`
    (`<=` → `≤`). Content-identical.
  - **#22** — the page's `eq` code block lists `eq(Field<T>)` before `eq(Select<…>)` for
    exposition; the source order is `eq(T)`, `eq(Select)`, `eq(Field)` (`Field.java:742/749/756`).
    Signatures are verbatim; only presentation order differs.
- **#34 caveat (pooling).** "jOOQ has no built-in pool; pooling delegated to HikariCP/Agroal/c3p0"
  — the _absence_ of a pool and the delegation to a `DataSource` are locally grounded
  (`DataSourceConnectionProvider`); the specific external pool product **names** are web/general
  knowledge, not in this tree.
- **#8 caveat (Java version).** The page cites the pinned `pom.xml` `<release>25</release>` as a
  build fact, not as jOOQ's runtime baseline. jOOQ ships multiple Java-baseline JARs (Java 8/11/17/21
  historically); those per-edition baselines are web-attested and deliberately not asserted from the
  parent pom.
