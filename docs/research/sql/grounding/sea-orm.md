## Discrepancies

- **#D1 (positioning taglines — grounded, not web).** The authoring brief anticipated the
  classic "async & dynamic ORM" and "built upon SQLx" slogans and suggested they might be
  web-attested. They are in fact **local**: `Cargo.toml:7` carries `description = "🐚 An
async & dynamic ORM for Rust"` (#1), and `src/docs.rs:3` states "Relying on [SQLx] …
  async support from day 1" (#7). The `sea-query` foundation is likewise local
  (`src/docs.rs:68` #8, `Cargo.toml:68` #4). No web fallback was needed for the
  positioning; the page grounds all of it in the tree.

- **#D2 (2.0-rc API surface).** The pinned tree is a **release candidate**, so the README
  and doctests show two overlapping APIs: the classic column/ActiveModel form
  (`fruit::COLUMN.cake_id.eq(2)`, `fruit::ActiveModel { id: Set(...), .. }`) and the new
  `2.0` dense form (`#[sea_orm::model]`, `HasOne`/`HasMany`, `ActiveModel::builder()`,
  `Entity::load().with(...)`). The page uses the classic form for its grounded examples
  (which back the `src/entity/column.rs` / `src/entity/active_value.rs` doctests) and
  notes the `2.0` additions separately. `src/entity/base_entity.rs:81-85` itself warns the
  `ActiveModelEx` API "is still evolving — its shape may change in minor releases" — the
  page flags this churn in Weaknesses. Not a contradiction; a versioning note.

- **#D3 (page links `master`, ledger pins `f0121e0`).** The deep-dive's reference links
  point at `github.com/SeaQL/sea-orm/blob/master/…` (consistent with the sibling pages,
  which link `master`/HEAD rather than a SHA). This ledger's line numbers are as-of the
  pinned `f0121e0`; `master` may drift. No factual impact — the quotes are verbatim at the
  pin.

## Unsourced / opinion

- Editorial framings — "the dynamic-ORM counterpoint to `Diesel`'s static builder", "the
  Rust counterpart to Go's `GORM` / `ent`", "async RAII, not a scoped effect", "typed-
  `Result` posture but coarse", "change-set closer to `Ecto`'s `Changeset` than to
  `Hibernate`'s dirty-checking", and the Strengths / Weaknesses / Key-design-decisions
  table — are author synthesis (◯), each consistent with the verified mechanics above. Not
  independently citable.
- **"No identity map / no implicit unit of work"** is an _absence_ argument: no such
  construct exists in `src/` (there is no session that snapshots loaded models and flushes
  a diff — `save`/`insert`/`update`/`delete` are explicit single calls, #38/#39). True by
  construction, framed against the [concepts ORM-patterns] ladder; not a positive quote.
- Weekly-download count (#52) and the dependents' current adoption (#53) are
  web/point-in-time; the sentences and the project table are in-tree, but the numbers and
  live usage are external. `0.1.0` ≈ 2021-08 is web/registry (the tree's oldest changelog
  entry is `0.1.1`, #55).
- SeaORM-X (commercial SQL Server) is mentioned in `README.md:436-438` (L) but is
  out-of-tree; the page notes it as an aside.
