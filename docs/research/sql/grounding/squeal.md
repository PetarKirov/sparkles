## Discrepancies

- **D1 (page corrects a possible over-reading of the brief) — error model.** The brief emphasises the
  indexed monad as "a very strong static guarantee." The page grounds that fully (claims 41–45) **and**
  adds the counterweight that Squeal's _error_ handling is conventional thrown exceptions, not a typed
  error channel (claims 56–57). This is a survey-relevant finding, not a contradiction of any source:
  `SquealException` is an ordinary `Exception` (`Session/Exception.hs:60`), thrown in `IO`. The page
  states this explicitly and contrasts it with `doobie`/`skunk`/`Effect TS` (plain-name, per the brief).

- **D2 ("no code generation" is an absence claim) — claim 32.** Verified by _absence_: the cabal's
  `exposed-modules` and the `src/` tree contain no introspection/codegen module (nothing analogous to
  `slick-codegen`/`sqlc`/`jOOQ`). Absence-of-feature cannot be quoted; it is asserted from the full
  module list and a negative grep. The page frames it as a deliberate code-first trade-off, not a defect.

- **D3 (metadata soft date) — claim 65.** "First release ≈2017" is web/soft: the `--depth 1` clone has
  no tags, so only the LICENSE copyright year (2017) grounds it locally. Marked ◯ on the page metadata
  ("≈2017 (web-soft)").
