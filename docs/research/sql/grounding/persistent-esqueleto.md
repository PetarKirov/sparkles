## Discrepancies

- **#D1 (brief correction — license).** The authoring brief said "MIT (check both LICENSEs)".
  persistent **is** MIT (`persistent.cabal:3`, `LICENSE` = MIT text), but **esqueleto is
  BSD-3-Clause** (`esqueleto.cabal:18` `license: BSD3`; `LICENSE` is the 3-clause BSD text,
  "Redistribution and use… Neither the name of Felipe Lessa…"). The page states both correctly
  (`persistent`: MIT; `esqueleto`: BSD-3-Clause). Not a page error — a corrected brief assumption.

- **#D2 (version nuance) — Experimental "default" version.** The page says the experimental
  `(:&)` syntax becomes the default in `4.0.0.0`. Sources give two figures: `Experimental.hs:7`
  says the module's `FROM` "will become the default syntax exported from the library in version
  `3.6.0.0`", while `Database/Esqueleto.hs:37` and `Internal.hs:168` say the definitions of
  `from`/`on` "will be replaced with those at the 4.0.0.0 version". The pinned tree **is**
  `3.6.0.0`, yet `Database.Esqueleto` still exports the legacy `from`/`on` with a `WARNING`
  pragma (`Database/Esqueleto.hs:43`) pointing at `4.0.0.0`. The page's "`4.0.0.0`" tracks the
  top-level module's own statement and the still-legacy default in this tree; the `3.6.0.0`
  figure refers narrowly to the `Experimental` module's internal `from`. Minor; both are quoted
  in-tree.

- **#D3 (heavy elision, not misquote) — Quasi module doc.** The `Database.Persist.Quasi:8-19`
  quote is condensed with "…" across several sentences ("the table name … followed by a list of
  fields … `fieldName FieldType`"). Every retained fragment is verbatim; the ellipses drop
  "in this case, TableName. It's" and "on the entity, which have the basic form". Fidelity-preserving.

- **#D4 (typographic) — CautiousMigration quote.** Source (`Migration.hs:82`) writes
  "operation is _unsafe_ - it might be destructive"; the page renders "unsafe — it might be
  destructive" (markdown `*emphasis*` dropped, ASCII hyphen→em-dash). Cosmetic.

- **#D5 (whitespace/markup normalization) — esqueleto goals quote (#10).** `Database/Esqueleto.hs:156`
  uses a double space ("to SQL. When") and `@esqueleto@` ddoc markup; the page normalizes to
  single space and backticks. Also the page cites both `Esqueleto.hs` and the cleaner `README.md:40-44`
  wording for the same three goals. Non-substantive.
