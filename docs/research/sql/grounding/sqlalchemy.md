  which link `main`/`master` rather than a SHA). This ledger's line numbers are as-of the
  pinned `5bf558a`; `main` may drift. No factual impact — the quotes are verbatim at the
  pin.

- **#D2 (pinned tree is `2.1.0b4`, page frames `2.0` as the stable line).** The checkout's
  `__version__` is a `2.1` beta (#5), but the page treats `2.0` as the "current stable
  line" and grounds the 2.0-era feature claims (typed Declarative, unified `select()`,
  `Query` legacy) in in-tree `versionadded 2.0` / `.. legacy::` notes (#26, #28, #57). Not
  a contradiction; the beta tree is a superset of 2.0. The page flags both versions in the
  metadata row.

- **#D3 (RST line-wrapping inside quotes).** The `README.rst` bullets wrap at ~40 columns,
  so the verbatim quotes (#1, #6-#12) preserve mid-sentence line breaks and double spaces
  exactly as the source has them (e.g. "the identity map, unit of work,\n and data mapper
  patterns. These patterns"). This is faithful copy-paste, not a transcription error; the
  double-spaces after periods are the source's own.
