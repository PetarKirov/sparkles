## Discrepancies

- **⚠ #21/#19 (nuance, not error).** The page draws a sharp line between the **hash form**
  (`where(name: x)`, a true out-of-band Arel bind parameter via `QueryAttribute`,
  `predicate_builder.rb:60-68`) and the **array form** (`where("age > ?", x)`, sanitized by
  `sanitization.rb` `quote_bound_value` → `connection.quote`, i.e. server-side escaping
  interpolated at build time). Both are injection-safe; the "out-of-band vs escaped-inline"
  distinction is accurate against the two code paths but is a **finer** statement than the
  `where` docstring itself makes (the doc only says the array form avoids injection
  attacks). Grounded, flagged as a code-read inference rather than a doc quote.

- **⚠ #32 (no-identity-map claim).** The page asserts Active Record has **no identity map**.
  The positive evidence is `persistence.rb:93-99` — finders `instantiate` a **new** object
  per row ("get new instances of the appropriate class for each record"). The _absence_ of
  an identity map is an architectural negative (nothing in the tree caches row→object), and
  the historical short-lived experimental identity map (Rails 3.0–3.1, later removed) is a
  web fact not present in this HEAD. The claim is sound but rests on absence + the
  instantiate-per-row doc, not on a single "there is no identity map" quote.
